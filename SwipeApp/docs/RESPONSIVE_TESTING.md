# Адаптивность приложения под разные размеры экранов

## Поддерживаемые размеры экранов

Приложение адаптировано под следующие популярные размеры экранов:

### Android устройства

- **1080×2400 (~360×800)** - около 11% устройств
- **412×915** - распространено среди среднеценовых Android-моделей
- **1440×3200 (~412×915)** - premium Android (Galaxy S-серия)

### iPhone устройства

- **375×812** - около 4% устройств
- **1170×2532 (~390×844)** - iPhone 14/15
- **1206×2622** - iPhone 16 Pro
- **1320×2868** - iPhone 16 Pro Max

## Адаптивная система

### Утилиты адаптивности (`src/utils/responsive.ts`)

1. **Определение типа устройства**

   ```typescript
   getDeviceType(); // возвращает: 'iphone-large', 'android-premium', etc.
   ```

2. **Процентные размеры**

   ```typescript
   wp(50); // 50% ширины экрана
   hp(30); // 30% высоты экрана
   ```

3. **Масштабирование**

   ```typescript
   scale(16); // масштабирует размер относительно базового дизайна
   ```

4. **Адаптивные размеры**
   - `fontSize` - размеры шрифтов (xs, sm, md, lg, xl, xxl, xxxl, huge)
   - `spacing` - отступы (xs, sm, md, lg, xl, xxl, xxxl, huge)
   - `deviceSizes` - размеры элементов

### Адаптивные компоненты

Все компоненты используют адаптивную систему:

- **UserCard** - карточки пользователей масштабируются под размер экрана
- **Button** - кнопки адаптируются по размеру и отступам
- **Input** - поля ввода с адаптивными размерами
- **Экраны** - все экраны используют адаптивные отступы и размеры

## Как тестировать

### В Expo/React Native

1. Используйте симулятор iOS с разными моделями iPhone
2. Используйте Android эмулятор с разными разрешениями
3. Тестируйте на реальных устройствах

### Проверка адаптивности

1. **Карточки пользователей** должны занимать оптимальную площадь экрана
2. **Кнопки** должны быть достаточно большими для удобного нажатия
3. **Текст** должен быть читаемым на всех размерах
4. **Отступы** должны выглядеть пропорционально
5. **Фотографии** должны корректно отображаться

### Ключевые проверки

#### Маленькие экраны (< 375px)

- [ ] Все элементы помещаются на экран
- [ ] Кнопки не слишком маленькие
- [ ] Текст читаемый

#### Средние экраны (375-414px)

- [ ] Оптимальное использование пространства
- [ ] Карточки выглядят пропорционально
- [ ] Удобная навигация

#### Большие экраны (> 414px)

- [ ] Нет излишне растянутых элементов
- [ ] Контент центрирован правильно
- [ ] Используется дополнительное пространство

## Настройка для новых размеров

Если нужно добавить поддержку нового размера экрана:

1. Обновите функцию `getDeviceType()` в `responsive.ts`
2. Добавьте новый случай в `getAdaptiveStyles()`
3. Протестируйте на всех экранах

## Базовые принципы

1. **Относительные размеры** - используем проценты и масштабирование
2. **Минимальные размеры** - кнопки не меньше 44px для удобства нажатия
3. **Читаемость** - минимальный размер шрифта 12px
4. **Пропорции** - сохраняем соотношения элементов
5. **Безопасные зоны** - учитываем SafeAreaView для современных устройств

## Отладка

Для отладки адаптивности добавьте в код:

```typescript
import { getDeviceType, screenDimensions } from "../utils/responsive";

console.log("Device type:", getDeviceType());
console.log("Screen dimensions:", screenDimensions);
```

Это поможет понять, как приложение определяет тип устройства.
